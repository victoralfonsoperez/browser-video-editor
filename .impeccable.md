# .impeccable.md — BrowserCut Design Context

## Design Context

### Users
General audience — anyone who needs a quick, capable browser-based video clip tool without installing software. They're not necessarily power users, but they're capable adults who appreciate efficiency. The context is typically: "I have a video file, I want to cut a clip, I don't want to sign up for anything or upload my footage anywhere." Privacy, speed, and trustworthiness matter more than feature depth.

### Brand Personality
Polished, purposeful, and calm. The brand in three words: **precise, trustworthy, frictionless**. Emotionally, the interface should make users feel competent — not overwhelmed by pro-tool complexity, not condescended to by consumer oversimplification. Think Linear, Vercel, Pitch: dark mode done with intent, not decoration. Dark mode here serves a real purpose (video editing context, focus, reduced eye strain) — not just aesthetic posturing.

### Aesthetic Direction
- **Visual tone**: Modern SaaS — refined, clean, high-contrast where it counts. Not flashy, not clinical.
- **References**: Linear (purposeful dark UI, strong information hierarchy), Vercel (crisp controls, minimal chrome), Pitch (clear primary actions, confident typography)
- **Anti-references**: Generic AI output (no cyan-on-dark, no purple gradients, no glowing glassmorphism cards). No Adobe-style visual density or menubar complexity. Not bubbly or hand-holdy.
- **Theme**: Dark mode only — intentional, not default
- **Accent**: The current lime-green (#c8f55a) is distinctive but risks reading as "neon on dark." Direction: use it with more restraint — functional signal rather than decorative glow.

### Design Principles
1. **Purposeful darkness** — the dark surface is a canvas for video, not a style choice. Surfaces should recede; content should lead.
2. **Progressive disclosure** — lead with the essential, reveal power through interaction. The primary action at each state should be unmistakable.
3. **Hierarchy through restraint** — not everything competes. One primary action per view. Supporting actions are discoverable, not prominent.
4. **Every word earns its place** — labels, hints, and copy should be short, specific, and useful. Remove anything a competent user could figure out.
5. **Calm confidence** — no anxiety-inducing clutter. The interface knows what it's doing, and the user should feel that.
